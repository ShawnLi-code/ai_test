## 1. 这个项目是干什么的

简单来说：**用 AI 模拟"客服打电话 → 用户接听 → 质检打分"的完整流程**，批量测试客服话术在 24 种不同用户反应下的表现。

它调用了四个 AI 模型协同工作：

| 模型 | 扮演角色 | 做什么 |
|------|----------|--------|
| **A 模型** | AI 客服 | 按照人设给用户打电话，介绍套餐升档活动 |
| **B 模型** | 质检员 | 通话结束后，检查 AI 客服有没有违规（按人设规定的审计标准打分） |
| **C 模型** | 模拟用户 | 按照人设 + 场景（如"质疑诈骗""情绪激动"）自然地回复客服 |
| **D 模型** | 工单生成 | 根据通话记录，自动生成结构化工单 JSON |

一次完整对话流程：A（客服）先说话 → C（用户）回复 → A 再回应 → ... 来回几轮后通话结束 → D 生成工单 → B 质检打分。

## 2. 准备工作：安装必需软件

你需要安装两个软件：**Git**（用于下载代码）和 **Python**（用于运行脚本）。

### 2.1 安装 Git

1. 打开浏览器，访问：**https://git-scm.com/download/win**
2. 页面会自动弹出下载，下载完成后双击运行安装程序
3. 安装过程中**一直点"Next"**，所有选项保持默认即可
4. 安装完成后，在桌面空白处右键，如果能看见 **"Open Git Bash here"** 就说明成功了

### 2.2 安装 Python

1. 打开浏览器，访问：**https://www.python.org/downloads/**
2. 点击页面中央黄色的 **"Download Python 3.x.x"** 按钮
3. 下载完成后双击运行安装程序
4. **关键步骤**：在安装界面的**最下方**，勾选 **"Add Python to PATH"**（一定要勾！）
5. 然后点击 **"Install Now"**，等它装完

### 2.3 验证安装

按下键盘上的 **Win 键**，输入 `cmd`，回车打开命令提示符（一个黑色窗口）。

在黑色窗口中依次输入以下两条命令（每输完一条按回车）：

```
git --version
python --version
```

如果都显示版本号（比如 `git version 2.x.x`、`Python 3.x.x`），说明安装成功。

## 4. 获取并配置 API Key

### 4.1 API Key 是什么

API Key 就像一把"钥匙"，让脚本能调用火山方舟（字节跳动的 AI 平台）上的模型。没有这把钥匙，脚本跑不起来。

### 4.2 获取 API Key

1. 打开浏览器，访问火山方舟控制台：**https://console.volcengine.com/ark/**
2. 登录你的火山引擎账号（没有的话需要先注册）
3. 在左侧菜单找到 **"API Key 管理"**（或类似入口）
4. 点击 **"创建 API Key"**，复制生成的那一串字符（类似 `c03439c9-xxxx-xxxx-xxxx-xxxxxxxxxxxx`）
5. **妥善保管**这串字符，不要发给别人

### 4.3 配置到本地

在代码文件夹中找到一个叫 `config.example.json` 的文件，用**记事本**打开它。

你会看到类似这样的内容：

```json
{
  "api_key": "",
  "base_url": "https://ark.cn-beijing.volces.com/api/v3",
  ...
}
```

**把 `"api_key": ""` 那一行改成**（注意保留引号和逗号）：

```json
"api_key": "把你复制的API Key粘贴到这里",
```

改好之后，在记事本里点 **文件 → 另存为**，文件名输入 `config.local.json`，保存到同一个文件夹里。

> `config.local.json` 已经被加入 `.gitignore`，不会被上传到 GitHub，不用担心泄露。

## 6. 理解 A/B/C/D 四个模型与人设

### 6.1 四个模型各用的是什么 AI

当前默认配置：

| 模型 | 底层 AI | 说明 |
|------|---------|------|
| A 模型 | **豆包（doubao-1-5-pro）** | 字节的模型，扮演 AI 客服 |
| B 模型 | **DeepSeek（deepseek-v3）** | 扮演质检员，做审计和打分 |
| C 模型 | **豆包（doubao-1-5-pro）** | 扮演用户，模拟各类用户反应 |
| D 模型 | **豆包（doubao-1-5-pro）** | 根据对话生成工单 |

> 如果你想换模型，修改 `config.local.json` 里的 `model_a`、`model_b` 等字段。但**一般不需要改**，默认配置已经是最佳组合。

### 6.2 什么是"人设"

人设就像是给 AI 的"角色设定稿"，告诉它在这场对话里应该怎么说话。人设文件放在 `prompts` 文件夹下：

```
prompts/
├── A/          ← AI客服的人设（A_v1.txt、A_v2.txt、A_v3.txt、A_v4.txt）
├── B/          ← 质检员的人设（B_v1.txt）
├── C/          ← 模拟用户的人设（C_v1.txt ~ C_v5.txt）
└── D/          ← 工单生成的人设（D_v1.txt）
```

- 文件名中的 `v1`、`v2` 代表**版本号**，数字越大越新。
- **目前最新版本**：A_v4、B_v1、C_v1 ~ C_v5、D_v1。

### 6.3 不同 C 人设代表什么类型的用户

| C 人设 | 用户画像 |
|--------|----------|
| **C_v1** | 30多岁普通上班族，理性，算账清晰就配合 |
| **C_v2** | 40多岁家庭主妇，对陌生来电警惕，需要耐心 |
| **C_v3** | 50多岁退休职工，听力理解稍慢，容易被绕晕 |
| **C_v4** | 20多岁大学生/初入职场，语速快，不耐烦 |
| **C_v5** | 40多岁小老板，急躁易怒，时间观念强 |

> 每个 C 人设文件首行的 `# 简介: xxx` 就是对用户性格的一句话概括。你可以在结果 Excel 里看到这个简介。

### 6.4 24 个测试场景

脚本内置了 24 个场景分类，覆盖客服工作中可能遇到的各种用户反应：

| 序号 | 场景 | 说明 |
|------|------|------|
| 1 | 正常接受 | 用户听懂了，愿意办理 |
| 2 | 直接拒绝 | 不感兴趣，直接说不需要 |
| 3 | 质疑诈骗 | 怀疑这是诈骗电话 |
| 4 | 价格异议 | 觉得套餐太贵 |
| 5 | 业务追问 | 反复追问细节 |
| 6 | 不方便接听 | 正在忙，没空聊 |
| 7 | 情绪激动 | 上来就发火/抱怨 |
| 8 | 非本人接听 | 接电话的不是机主 |
| 9 | 犹豫观望 | 感兴趣但下不了决定 |
| 10 | 打断测试 | 频繁打断客服说话 |
| 11 | 投诉威胁 | 扬言要投诉 |
| 12 | 法律维权 | 搬出法律条款来说话 |
| 13 | 隐私担忧 | 担心个人信息泄露 |
| 14 | 诱导陷阱 | 试图诱导客服说违规的话 |
| 15 | 噪音/听不清 | 周围环境嘈杂，信号不好 |
| 16 | 沉默/不回应 | 不说话，晾着客服 |
| 17 | 反复切换态度 | 一会要办一会不办 |
| 18 | 老人/理解困难 | 年纪大，理解慢 |
| 19 | 要求转人工 | 不想跟AI聊，要真人 |
| 20 | 套取信息 | 试图套取员工或系统内部信息 |
| 21 | 恶意骚扰反制 | 反过来骚扰客服 |
| 22 | 比较竞品 | 拿联通/电信的套餐来比 |

## 7. 运行测试

### 7.1 打开终端

按下 **Win 键**，输入 `cmd`，回车打开命令提示符。

先进入代码目录：

```
d:
cd D:\ai_test
```

### 7.2 最简单的运行方式

什么都不指定，使用配置文件里的默认值：

```
python ai_test_script.py
```

这会用 `config.local.json` 里配置的默认 A/B/C 人设跑全量 24 个场景。

### 7.3 指定人设版本运行（最常用）

每次测试时通过命令指定要测哪些人设版本：

```
python ai_test_script.py --persona A_v4 B_v1 --persona-c-list C_v1 C_v2 C_v3 C_v4 C_v5
```

这条命令的意思是：
- `--persona A_v4 B_v1`：A 模型用 v4 版本，B 模型用 v1 版本
- `--persona-c-list C_v1 C_v2 C_v3 C_v4 C_v5`：C 模型全部 5 个人设版都跑一遍

### 7.4 调试时快速跑（推荐）

如果只是想快速检查人设能不能跑通，加 `--limit 3 --skip-audit`：

```
python ai_test_script.py --persona A_v4 B_v1 --persona-c-list C_v1 C_v2 --limit 3 --skip-audit --max-rounds 4
```

- `--limit 3`：只跑前 3 个场景
- `--skip-audit`：跳过 B 模型的质检（省时间）
- `--max-rounds 4`：每通对话最多 4 轮

### 7.5 并发参数说明

| 参数 | 默认值 | 作用 | 建议 |
|------|--------|------|------|
| `--concurrency` | 10 | 同时跑多少个场景 | 调试用 3~5，全量用 10~20 |
| `--persona-concurrency` | 2 | 同时跑多少个 C 人设 | C 人设有几个设几个 |
| `--api-concurrency` | 30 | API 调用的上限 | 一般 30~50，太高容易限流 |

高并发示例（跑得快，但也更容易被限流）：

```
python ai_test_script.py --persona A_v4 B_v1 --persona-c-list C_v1 C_v2 C_v3 C_v4 C_v5 --concurrency 20 --persona-concurrency 5 --api-concurrency 50
```

低并发示例（稳，适合长时间运行的正式测试）：

```
python ai_test_script.py --persona A_v4 B_v1 --persona-c-list C_v1 C_v2 C_v3 C_v4 C_v5 --concurrency 5 --persona-concurrency 2 --api-concurrency 20
```

### 7.6 等待运行完成

运行过程中，终端会不断刷出类似这样的进度信息：

```
[C_v1][1/24] 正常接受: ✓ 合格 (违规0项)
[C_v1][2/24] 直接拒绝: ✓ 合格 (违规0项)
[C_v2][3/24] 质疑诈骗: ✗ 不合格 (违规2项)
...
```

- `✓ 合格`：质检通过，对话没问题
- `✗ 不合格`：质检发现问题，需要查看具体违规项
- 如果出现 `[异常]`：网络问题或 API 报错，脚本会自动重试

最后会输出汇总：

```
============================================================
  多C人设测试完成: 共24个分类，5个C人设
  C_v1: 合格 22  不合格 2  跳过质检 0  异常 0  耗时 2分17秒
  C_v2: 合格 23  不合格 1  跳过质检 0  异常 0  耗时 2分6秒
  ...
  本次测试总耗时: 3分24秒
============================================================
```

## 8. 查看测试结果

### 8.1 结果文件在哪

测试完成后，结果保存在 `results` 文件夹里，有两个地方：

| 路径 | 说明 |
|------|------|
| `results\latest\` | **最新一次**的测试结果（每次跑都会覆盖） |
| `results\runs\20260525_103431_A_v4_B_v1_...\` | **带时间戳的历史记录**（每次跑都会保留，不会丢失） |

### 8.2 打开结果 Excel

直接双击 `results\latest\AI外呼多C测试结果_A_v4_B_v1.xlsx` 即可打开。

### 8.3 Excel 里有什么

每个 Excel 文件包含多张工作表（Sheet），底部标签页可以切换：

- **每个 C 人设一张独立工作表**：比如 `C_v1测试结果`、`C_v2测试结果` 等
- 每张工作表里列出了该人设 × 24 个场景的完整测试结果

**表格每列的含义：**

| 列 | 含义 |
|----|------|
| 序号 | 场景编号 |
| 场景分类 | 测试的场景名称 |
| A(客服)对话记录 | AI 客服的完整发言 |
| AI客服轮数 | 客服说了几轮话 |
| C(用户)应答记录 | 模拟用户的完整发言 |
| 用户轮数 | 用户说了几轮话 |
| 质检合格 | ✓ 或 ✗ |
| 不合格项 | 如果有违规，这里列出具体违规内容 |
| 工单内容 | D 模型生成的工单 JSON |

**颜色标记：**
- **浅红色底色**的行 = 质检不合格，需要重点关注
- **浅橙色底色**的行 = 运行异常（网络错误等）

### 8.4 快速定位问题

1. 打开 Excel，先看底部标签页，切换不同 C 人设
2. 扫一眼"质检合格"列，找到 ✗ 的行
3. 红色行看"不合格项"列了解具体违规内容
4. 对照"A(客服)对话记录"和"C(用户)应答记录"来复盘问题

## 9. 常用命令速查

以下命令假设你已在终端中进入了代码目录（`d:` 然后 `cd D:\ai_test`）。

```powershell
# 全量测试（默认人设）
python ai_test_script.py

# 全量测试（指定A、B版本 + 全部C人设）
python ai_test_script.py --persona A_v4 B_v1 --persona-c-list C_v1 C_v2 C_v3 C_v4 C_v5

# 只跑单个C人设
python ai_test_script.py --persona A_v4 B_v1 --persona-c C_v1

# 只跑前5个场景（调试用）
python ai_test_script.py --persona A_v4 B_v1 --persona-c C_v1 --limit 5

# 快速冒烟测试（跳过质检，省时间）
python ai_test_script.py --persona A_v4 B_v1 --persona-c C_v1 --limit 3 --skip-audit --max-rounds 4

# 跳过工单生成（省掉D模型调用）
python ai_test_script.py --persona A_v4 B_v1 --persona-c C_v1 --skip-ticket

# 随机抽10个场景快速回归
python ai_test_script.py --sample-size 10 --sample-seed 20260523

# 回归对比（新旧版本对比）
python ai_test_script.py --regression --old A_v3 B_v1 --new A_v4 B_v1 --persona-c C_v1
```

## 10. 常见问题

### Q1：提示 "未设置api_key"

**原因**：没有配置 API Key，或者 `config.local.json` 文件位置不对。

**解决**：
1. 确认 `config.local.json` 文件在代码目录下（和 `ai_test_script.py` 同一层）
2. 用记事本打开确认 `api_key` 字段已填写

### Q2：提示 "'python' 不是内部或外部命令"

**原因**：安装 Python 时没勾选 "Add Python to PATH"。

**解决**：重新运行 Python 安装程序，选择 **"Modify"**，在后续界面勾选 **"Add Python to PATH"**，完成后再重新打开命令提示符。

### Q3：提示 "Connection error" 或 "超时"

**原因**：网络不稳定，或并发开太高被限流了。

**解决**：降低并发参数重新跑：

```
python ai_test_script.py --persona A_v4 B_v1 --persona-c-list C_v1 C_v2 C_v3 C_v4 C_v5 --concurrency 3 --persona-concurrency 1 --api-concurrency 10
```

### Q4：输出 Excel 里中文乱码

**原因**：PowerShell 默认编码问题。

**解决**：使用命令提示符（cmd）而不是 PowerShell 来运行脚本。或者在 PowerShell 中先执行：

```
[Console]::OutputEncoding = [Text.Encoding]::UTF8
```

### Q5：全量测试要跑多久

**参考时间**：
- 全量 24 场景 × 5 个 C 人设（共 120 通对话）：约 **3~5 分钟**
- 单 C 人设（24 通对话）：约 **1~2 分钟**
- 冒烟测试（3 场景，跳过质检）：约 **20~40 秒**

实际时间取决于网络速度和模型响应速度。

### Q6：想把结果发给别人看

直接把 `results\runs\` 文件夹下对应时间戳的整个文件夹打包发过去，或者把 `results\latest\` 里的 Excel 文件发过去即可。

### Q7：想修改人设内容

1. 用记事本打开 `prompts\A\`（或 B/C/D）下的 `.txt` 文件
2. 直接修改内容，保存
3. **注意**：如果是迭代现有版本，建议复制一份改文件名（如 `A_v4.txt` → `A_v5.txt`），保持旧版本可回溯
4. 修改后跑一次冒烟测试验证改动是否生效

### Q8：怎么更新到最新代码

在代码目录下打开命令提示符，输入：

```
git pull
```

这会把 GitHub 上的最新改动同步到本地。
